Setting up a local AI provider
1Install Ollama
curl -fsSL https://ollama.com/install.sh | sh
Or download the installer from ollama.com for macOS / Windows.
2Pull a model
ollama pull llama3.2
Other good options: mistral gemma2:9b phi3
3Start the server
ollama serve
4Set in Truey
Use endpoint http://localhost:11434 — no API key needed.
Ollama runs entirely on your device. Queries never leave your machine when using a local model.
1Build or download llama.cpp
git clone https://github.com/ggml-org/llama.cpp && cd llama.cpp && make
Pre-built binaries are available in the GitHub releases for macOS, Linux, and Windows.
2Download a GGUF model
Get a .gguf file from Hugging Face (e.g. Llama-3.2-3B-Instruct-Q4_K_M.gguf).
3Start the server
./llama-server -m ./your-model.gguf --port 8080
4Set in Truey
Use endpoint http://localhost:8080 — no API key needed.
llama.cpp gives you lower-level control and is great for GPU acceleration with CUDA or Metal.
